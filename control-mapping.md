# Security Control Mapping

This document maps identified risks to recommended security controls
based on common industry cybersecurity frameworks.

The mapping demonstrates how organisations translate identified risks
into actionable security controls aligned with best practices.

---

## Risk to Control Mapping

| Risk | Recommended Control | Framework Reference |
|-----|-----|-----|
| Phishing attacks | Security awareness training | NIST PR.AT |
| Malware infections | Endpoint protection and anti-malware solutions | NIST PR.IP |
| Weak passwords | Strong password policy and MFA | ISO 27001 A.9 |
| Unauthorized access | Identity and access management controls | ISO 27001 A.9 |
| Unpatched systems | Patch management program | NIST PR.IP |
| Data breaches | Data protection and encryption | ISO 27001 A.10 |

---

## Explanation

Security controls are implemented to reduce the likelihood or impact
of cybersecurity risks.

By aligning controls with recognised frameworks, organisations ensure
their security practices follow internationally accepted standards.

---

## Frameworks Referenced

NIST Cybersecurity Framework  
ISO 27001 Information Security Management Standard
